# 디자인 시스템 — 대진대 택시 타자

> 대진대 택시 타자 웹앱의 일관된 UI/UX 제공을 위해 색상, 폰트, 컴포넌트, 모바일 가독성 규칙을 정의합니다.

## 1. 디자인 키워드

- **분위기**: 신뢰감 있는, 깔끔한, 직관적인, 모바일 특화
- **참고 서비스**: 토스(Toss - 미니멀한 UI와 명확한 타이포그래피), 당근(Daangn - 커뮤니티 태그 기반 피드)
- **피해야 할 느낌**: 너무 화려하거나 복잡한 그라데이션, 작은 텍스트가 빽빽한 화면, 터치가 불편한 좁은 버튼 영역

## 2. 색상

| 역할 | 색상 코드 | 용도 |
| --- | --- | --- |
| **Primary** | `#003893` (Daejin Deep Blue) | 주요 동작 버튼, 로고 강조, 핵심 헤더 |
| **Primary Light** | `#EBF2FF` | 주요 강조 텍스트 배경, 수락 버튼 배경 등 |
| **Accent (Warning)** | `#FF3B30` (Red) | 거절, 탑승 상태 취소, 경고 메시지 |
| **Accent (Success)** | `#34C759` (Green) | 정산 완료, 매칭 성공, 탑승 완료 상태 |
| **Background** | `#F2F4F7` (Light Grey) | 앱 전체 외곽 배경 |
| **Surface** | `#FFFFFF` (White) | 입력 창, 카드 컴포넌트, 채팅 버블(나) |
| **Surface Grey** | `#F8F9FA` | 상대방 채팅 버블, 비활성 입력창 |
| **Text Primary** | `#1F2937` (Dark Grey) | 주요 폰트 색상, 타이틀 |
| **Text Secondary**| `#4B5563` (Medium Grey) | 부제목, 메타 정보(시간, 인원 등) |
| **Border** | `#E5E7EB` | 카드 구분선, 입력 폼 경계선 |

## 3. 폰트와 글자 크기

- **기본 폰트**: `Pretendard, -apple-system, BlinkMacSystemFont, system-ui, sans-serif`
- **글자 크기 규칙**:
  - **Title 1 (대제목)**: 20px / Bold (주요 화면 타이틀)
  - **Title 2 (중제목)**: 16px / SemiBold (카드 타이틀, 섹션 제목)
  - **Body 1 (본문 주요)**: 15px / Regular (입력값, 채팅 내용)
  - **Body 2 (본문 부차)**: 13px / Regular (시간, 태그, 부가 정보)
  - **Button Text**: 16px / SemiBold (터치용 메인 버튼 글자)

## 4. 컴포넌트 규칙

| 컴포넌트 | 규칙 |
| --- | --- |
| **버튼 (Button)** | 높이 `48px` 이상 확보 (터치 미스 방지), 둥근 모서리(`12px`), 핵심 행동(Primary)은 단색 채움, 보조 행동은 테두리/연한 색상 적용 |
| **카드 (Card)** | 흰색 배경(`Surface`), 모서리 둥글기 `16px`, 아주 옅은 그림자(`shadow-sm`) 또는 테두리(`border`) 적용 |
| **입력창 (Input)** | 높이 `48px` 이상, Placeholder와 입력 시 Font Color 구분 명확화, 라벨 배치 필수 |
| **태그 (Tag)** | 성별 및 상태 정보를 위해 `border-radius: 6px` 적용, 성별(블루/핑크/그레이), 상태(모집/탑승/정산에 맞는 컬러매칭) |
| **네비게이션 헤더** | 상단에 뒤로가기 버튼과 페이지 제목 표시 (터치 영역 `48px` 확보) |

## 5. 모바일 기준

- **최대 폭 (Max Width)**: `480px` 고정 및 화면 중앙 정렬 (`mx-auto min-h-screen bg-white shadow-lg`)
- **터치 영역**: 모든 버튼과 인터랙션 요소는 최소 `48px` 높이/너비를 확보하여 한 손 조작성 향상
- **여백**: 좌우 기본 패딩 `16px` 적용
- **하단 고정**: 글쓰기 완료, 매칭 참여 신청 등 가장 핵심적인 동작은 화면 최하단에 고정 플로팅 버튼(Sticky Bottom) 배치
