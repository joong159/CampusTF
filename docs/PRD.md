# PRD — 대진대 택시 타자

> 대진대 택시 타자 웹앱의 제품 요구사항 문서입니다. MVP 단계의 개발 범위와 목표, 핵심 기능을 정의합니다.

## 1. 개요

| 항목 | 내용 |
| --- | --- |
| **제품명** | 대진대 택시 타자 (Daejin Taxi Taja) |
| **한 줄 정의** | 대진대 학생들의 교통비 절감 및 대기시간 단축을 위한 실시간 택시 동승 매칭 플랫폼 |
| **플랫폼** | 모바일 우선 웹앱 (최대 폭 480px) |
| **핵심 사용자** | 버스 배차 대기가 힘들거나 혼자 택시를 타기 부담스러운 대진대학교 학생 전체 |

## 2. 목표

1. **신속한 매칭**: 등하교 시간대 대기 시간을 최소화하고, 단 몇 번의 터치로 동승 방 참여 유도
2. **안전성 확보**: 학번 정보 입력 및 동성(남자만/여자만) 매칭 옵션을 통해 신뢰도 향상
3. **정산 투명성**: N분의 1 자동 정산 가이드 및 계좌 고정 공지를 통해 정산 피로도 해결

## 3. 핵심 기능

### F1. 사용자 인증 및 가입
- **목적**: 대진대 학생 식별 및 성별 필터링용 데이터 확보
- **사용자 행동**: 이메일, 패스워드, 학번(Student ID), 성별(남/여)을 입력하여 가입 및 로그인 수행
- **결과**: Supabase Auth와 매칭되는 사용자 프로필 생성 완료

### F2. 실시간 택시 팟 탐색 및 필터링
- **목적**: 본인의 현재 경로와 맞는 팟(방)을 신속하게 추천하고 목록 제공
- **사용자 행동**: 홈 화면 상단에 나의 등하교 경로(출발지, 목적지)를 입력하고, 나와 딱 맞는 추천 팟 확인 및 출발 시간순 모집글 목록 탐색
- **결과**: 경로가 유사한 방을 찾거나, 전체 방 목록에서 성별 조건(누구나/남자만/여자만)과 잔여석(예: 2/4) 확인 가능

### F3. 방 만들기 및 신청자 수락/거절 (신청자 관리)
- **목적**: 방장의 주도하에 동승자를 안전하게 모집하고 통제
- **사용자 행동**: 방장이 출발지, 목적지, 출발 시간, 모집 인원(최대 4인), 성별 옵션(누구나/동성끼리만)을 입력하여 방 개설. 다른 사용자가 [참여 신청]을 누르면 방장이 신청자 목록을 확인하고 [수락] 또는 [거절] 처리
- **결과**: 수락된 인원은 채팅방에 입장할 권한을 획득하고 방의 참여 인원 수 증가

### F4. 실시간 채팅 및 탑승 상태 관리 & N분의 1 정산 가이드
- **목적**: 동승자 간 소통 채널 확보 및 실제 운행 단계별 상태 관리와 편리한 정산
- **사용자 행동**:
  - 방장 및 수락된 인원이 Supabase Realtime 기반 채팅방에서 실시간 대화 진행
  - 채팅방 상단에 방장의 계좌 정보/카카오페이 송금 링크 및 탑승 후 송금 유도 공지 배너 고정
  - 방장이 탑승 상태 변경: `모집 중` ➔ `모집 마감` ➔ `탑승 중` ➔ `정산 요청`
  - `정산 요청` 상태 변경 시, 방장이 총 택시 요금을 입력하면 자동으로 **N분의 1 정산 금액**을 계산하여 참여자 화면에 띄워줌
- **결과**: 투명한 정산 완료 및 탑승 여정 마무리

## 4. 범위

### 이번 버전에 만든다
- F1 (이메일/패스워드/학번/성별 회원가입 및 로그인)
- F2 (경로 입력, 추천 팟 표시, 전체 목록 카드 정렬 - 성별 태그, 인원 표시 포함, 방만들기 Floating [+] 버튼)
- F3 (방 개설, 방장용 신청자 관리 화면 - 수락/거절 인터랙션)
- F4 (실시간 채팅, 탑승 상태 관리 버튼, 정산 계좌번호 안내 및 N분의 1 자동 계산 UI)

### 이번 버전에 만들지 않는다
- 카카오/네이버 소셜 로그인 (기본 Supabase 이메일 가입만 제공)
- 실시간 지도 경로 API 연동 및 택시 위치 GPS 추적 (출발/도착지는 텍스트 및 대표 거점 선택으로 대체)
- 인앱 간편결제 시스템 (계좌이체 및 카카오페이 송금 링크 이동으로 대체)

## 5. 데이터

| 데이터 | 필드 예시 | 설명 |
| --- | --- | --- |
| **사용자 (profiles)** | `id`(UUID), `email`, `student_id`, `gender` (남/여) | 사용자 식별 및 필터용 프로필 데이터 |
| **방 (rooms)** | `id`(UUID), `created_by`, `departure`, `destination`, `departure_time`, `capacity`, `gender_filter` (anyone/same_gender), `status` (recruiting/closed/riding/settlement), `total_fare`, `bank_account`, `kakaopay_url` | 택시 팟 정보 및 상태 |
| **신청자 (applicants)** | `id`(UUID), `room_id`, `user_id`, `status` (pending/accepted/rejected) | 방 참여 신청 현황 |
| **채팅 (messages)** | `id`(UUID), `room_id`, `user_id`, `content`, `created_at` | 실시간 소통 메시지 |

## 6. 권한

- **로그인 없이 가능한 것**: 로그인 및 가입 화면 진입
- **로그인해야 가능한 것**: 홈 화면 조회, 내 경로 입력, 방 생성, 방 참여 신청, 실시간 채팅, 상태 확인
- **방장만 가능한 것**: 신청자 수락/거절, 탑승 상태 관리(`모집 마감` / `탑승 중` / `정산 요청`), 정산 정보(계좌/총요금) 설정
- **수락된 참여자만 가능한 것**: 해당 방의 채팅방 입장 및 메시지 송수신, 정산 가이드 확인

## 7. 완료 기준

- [ ] 모바일 뷰(최대 480px, 중앙 정렬) 및 48px 이상의 터치 영역이 완벽히 적용된다.
- [ ] Supabase Auth 기반으로 성별 필드와 학번을 포함하여 회원가입/로그인이 정상 작동한다.
- [ ] 추천 팟 및 모집 카드 리스트가 실시간 연동되며, 방 개설이 가능하다.
- [ ] 방장의 신청자 수락/거절 처리에 따라 채팅방 입장 권한이 실시간 변경된다.
- [ ] 실시간 채팅 및 탑승 상태 변경(`정산 요청` 시 N분의 1 계산기 작동) 기능이 동작한다.
