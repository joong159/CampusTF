# 기능정의서 — 대진대 택시 타자

> 대진대 택시 타자 서비스의 상세 기능별 입력값, 동작 및 예외 처리를 규정합니다.

## 기능 목록

| ID | 기능 | 우선순위 | 상태 |
| --- | --- | --- | --- |
| **F1** | 사용자 가입 및 인증 | 높음 | 준비 |
| **F2** | 매칭 홈 및 검색/필터링 | 높음 | 준비 |

## F1. 사용자 가입 및 인증
- **목적**: 대진대 학생 식별 및 성별 확인을 통한 신뢰성 있는 매칭 제공
- **사용자**: 신규 및 기존 대진대 학생
- **진입 화면**: 로그인 및 회원가입 화면

### 입력값
- 이메일: 필수 / 이메일 형식 준수 (예: `@daejin.ac.kr` 도메인 권장하나 테스트 편의를 위해 모든 이메일 허용)
- 비밀번호: 필수 / 6자 이상
- 학번: 필수 / 숫자로 구성된 8자~10자 학번 (예: `20201234`)
- 성별: 필수 / 남성 또는 여성 택일

### 동작
1. 사용자가 정보를 입력하고 [회원가입] 버튼을 누른다.
2. Supabase Auth를 통해 이메일/비밀번호 가입을 진행한다.
3. 가입 성공 시 사용자 ID(`uid`)와 매칭되는 정보를 `profiles` 테이블에 저장한다 (학번, 성별 기록).
4. 가입 완료 후 자동으로 로그인 처리되어 홈 화면으로 이동한다.

### 예외 상황
- 필수값 누락: "모든 항목을 입력해주세요." 안내 메시지 노출
- 비밀번호 자수 부족: "비밀번호는 6자리 이상이어야 합니다." 경고
- 기존 가입 이메일: "이미 가입된 이메일 주소입니다." 안내

## F2. 매칭 홈 및 검색/필터링
- **목적**: 현재 등하교 경로에 맞는 동승 팟을 신속히 매칭하고 전체 목록 탐색
- **사용자**: 로그인된 모든 사용자
- **진입 화면**: 홈 화면

### 입력값
- 출발지: 선택형 / `대진대역`, `포천터미널`, `대진대 정문`, `의정부역`, `직접 입력`
- 목적지: 선택형 / `대진대 정문`, `대진대역`, `포천터미널`, `의정부역`, `직접 입력`

### 동작
1. 사용자가 상단 경로 입력 영역에서 출발지와 목적지를 선택 또는 입력한다.
2. '나와 딱 맞는 추천 팟' 컴포넌트에 해당 경로와 일치하는 방을 필터링하여 노출한다.
3. 하단에 출발 시간순으로 정렬된 전체 모집 글 리스트 카드를 보여준다.
4. 각 카드에는 `출발/목적지`, `출발시간`, `현재 인원/모집 인원 (예: 2/4)`, `성별 필터 태그 (누구나/남자만/여자만)`를 표시한다.

## F3. 방 개설 및 신청자 관리 (수락/거절)
- **목적**: 방장의 세부 조건 설정 및 동승 참여자 통제
- **사용자**: 방장 (개설자) 및 신청 학생
- **진입 화면**: 홈 화면 우측 하단 [+] 플로팅 버튼 클릭 ➔ 방 만들기 화면

### 입력값
- 출발지, 목적지: 필수 (직접 입력 또는 퀵 선택)
- 출발시간: 필수 (시간 선택)
- 모집인원: 필수 / 2명 ~ 4명 중 선택
- 성별필터: 필수 / '누구나', '동성끼리만' 선택
- 계좌정보: 필수 / 은행명 및 계좌번호 입력 (예: `국민은행 123-456-7890`)
- 카카오페이 송금 링크: 선택 / 카카오페이 송금용 URL 주소

### 동작
1. 방장이 필수 정보를 입력한 후 [방 개설하기]를 누른다.
2. `rooms` 테이블에 상태 `recruiting`(모집 중)으로 데이터를 저장한다.
3. 일반 사용자가 홈 화면에서 해당 방 카드에 있는 [참여 신청]을 누르면, `applicants` 테이블에 `status: pending`으로 저장된다.
4. 방장은 자신의 방 카드나 관리 탭에서 신청자 목록(학번, 성별)을 확인하고 **[수락]** 또는 **[거절]**을 선택한다.
5. **[수락]** 시, `applicants` 상태가 `accepted`로 변경되고 방의 현재 참여 인원수가 증가한다.
6. **[거절]** 시, 상태가 `rejected`로 변경되어 신청 불가 상태가 된다.

## F4. 실시간 채팅 및 탑승 상태 변경 & 정산 가이드
- **목적**: 실시간 대화 및 운행 여정 완료에 따른 자동 N/1 정산
- **사용자**: 방장 및 수락된 탑승자
- **진입 화면**: 수락 완료 후 홈 화면 카드 또는 네비게이션을 통해 진입하는 채팅방

### 입력값
- 채팅 메시지 텍스트: 1자 이상
- 실제 총 택시 요금: 필수 (`정산 요청` 단계에서 방장이 입력)

### 동작
1. **실시간 대화**: Supabase Realtime을 통해 동승자 간 실시간 메시지 송수신이 이루어진다.
2. **공지 고정 배너**: 채팅창 상단에 방장의 은행 계좌 정보 및 카카오페이 링크가 고정되어 표시된다.
3. **상태 변경 (방장 전용)**:
   - 모집 마감: 방장이 `[모집 마감]` 버튼을 누르면 상태가 `closed`로 변경되며 추가 신청이 불가능해진다.
   - 탑승 중: 방장이 `[탑승 중]` 버튼을 누르면 상태가 `riding`으로 변경된다.
   - 정산 요청: 방장이 `[정산 요청]` 버튼을 누르면, **총 택시 요금 입력 모달**이 발생한다.
4. **자동 N분의 1 정산**:
   - 방장이 총 요금을 입력하면 (예: `8,000원`), 시스템은 **`총 요금 ÷ (방장 + 수락된 인원수)`**를 자동으로 계산한다 (예: 4명 탑승 시 인당 `2,000원`).
   - 채팅방 상단 배너에 **"인당 송금액: 2,000원"** 가이드와 계좌 정보가 강조 노출되어 송금을 유도한다.
